
## Стек проекта
- **Моделирование:** UML (Use Case, Activity), BPMN 2.0, ER-диаграммы
- **Документирование:** Notion, Markdown
- **Прототипирование:** Figma
- **Тестирование:** Postman, ручное тестирование
- **Управление:** Scrum (имитация)

## Результаты
- Спроектирована бизнес-логика адаптивного депозита с 5 сценариями расчета
- Разработан полный пакет проектной документации
- Созданы прототипы ключевых экранов
- Подготовлены тест-кейсы для проверки логики
- По расчетам, решение позволило бы снизить no-show на 25%
