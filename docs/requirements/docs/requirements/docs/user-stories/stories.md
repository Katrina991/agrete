# Пользовательские истории (User Stories)

## Эпик 1: Бронирование столика

### US-01: Расчет депозита
**Как** гость, который планирует визит,
**Я хочу** видеть точную сумму депозита при выборе времени,
**Чтобы** понимать итоговую стоимость и принять решение.

**Критерии приемки:**
- При выборе пятницы 19:00 в топ-ресторане отображается депозит 30%
- При выборе вторника 15:00 отображается депозит 10%
- Рядом с суммой есть иконка "i" с пояснением правил расчета

### US-02: Оплата депозита
**Как** гость,
**Я хочу** оплатить депозит онлайн картой,
**Чтобы** гарантированно получить столик.

**Критерии приемки:**
- После выбора времени открывается форма оплаты
- Можно оплатить картой любого банка
- После оплаты приходит email с подтверждением
- При ошибке оплаты бронь не создается

### US-03: Отмена брони
**Как** гость, чьи планы изменились,
**Я хочу** отменить бронь в личном кабинете,
**Чтобы** не потерять деньги и освободить столик.

**Критерии приемки:**
- В личном кабинете есть кнопка "Отменить бронь"
- При отмене показывается сумма возврата
- Возврат происходит на карту в течение 3-5 дней

## Эпик 2: Управление бронями

### US-04: Просмотр броней
**Как** администратор ресторана,
**Я хочу** видеть все брони на сегодня в одном списке,
**Чтобы** планировать загрузку зала.

**Критерии приемки:**
- В админ-панели есть список всех броней
- Брони сортируются по времени
- Оплаченные брони помечены зеленым

### US-05: Настройка правил депозита
**Как** владелец ресторана,
**Я хочу** настраивать процент депозита для разных дней,
**Чтобы** гибко управлять заполняемостью.

**Критерии приемки:**
- В настройках есть таблица "День недели — процент"
- Можно задать % для пятницы, субботы отдельно
- Можно задать % для будних дней отдельно
