# Модуль «Умное бронирование» для агрегатора столиков

## О проекте
Учебный проект по системному анализу. Цель — разработать модуль адаптивного депозита для сервиса бронирования столиков (аналог Яндекс.Столы, R-Keeper). Проект выполнен в рамках курсовой работы в Московском технологическом институте.

## Проблема
Рестораны теряют до 15-20% выручки из-за неявок гостей (no-show). При этом фиксированный депозит не решает проблему: для популярного времени он слишком мал, для непикового — отпугивает гостей.

## Решение
Разработка модуля «умного» депозита, который динамически рассчитывает сумму предоплаты в зависимости от:
- дня недели
- времени бронирования
- популярности ресторана
- истории посещений гостя

## Моя роль
Системный аналитик (полный цикл):
- Сбор и анализ требований
- Проектирование бизнес-логики
- Разработка UML-диаграмм и BPMN-схем
- Написание пользовательских историй
- Создание спецификации требований (SRS)
- Прототипирование интерфейсов
- Тестирование сценариев
